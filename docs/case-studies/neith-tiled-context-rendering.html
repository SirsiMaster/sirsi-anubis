Tiled Context Rendering

Case Study 020 — A GPU-Inspired Approach to AI Development Efficiency

ADR-013 Neith — The Weaver Pantheon v0.11.0

The Problem

Every developer using an AI coding assistant hits the same wall. You start a session, load your project context — architecture docs, decision records, coding standards, project memory — and begin working. Then context pressure builds. You clear. And now you reload everything. The same 254,000 tokens of project knowledge, paid for again. And again. Five times a day.

This is the re-read tax. It's invisible on any single session. It's devastating over a project lifecycle.

Pantheon's Thoth module already eliminated 97% of the re-read tax by compressing project state into ~3,000 tokens of persistent memory. But the first session of every sprint still loaded the full canon. And every Ra-deployed autonomous agent loaded it on every sprint. We needed to load context intelligently — not less context, but the right context.

The Insight: GPUs Solved This Decades Ago

GPU RenderingAI Context
Scene geometryFull project canon (ADRs, docs, memory, changelog)
Camera viewportCurrent development scope
Tile visibility testRelevance scoring against scope keywords
Fragment shadingToken consumption by the LLM
Deferred tilesManifest of available but unloaded documents

Modern GPUs use tiled deferred rendering. The full scene is loaded — nothing discarded. A visibility test determines what's on screen. Only visible fragments get expensive shading. Everything else is deferred but available.

We applied the same principle to context management: nothing is discarded, but only relevant content is rendered into the prompt.

The Pipeline

LoadCanon()    → CanonContext     [geometry pass — all data loaded]
ChunkCanon()   → []CanonChunk    [tiling — semantic units]
ScoreChunks()  → []ScoredChunk   [z-test — multi-signal scoring]
TilePrompt()   → TileResult      [shading — fill budget, build manifest]
WeaveScope()   → prompt          [compositing — sections + manifest]

Why Tiling Makes Pantheon Compelling

Without tiling, Pantheon already delivers 2.3x the productive output of vanilla AI assistance. Thoth's persistent memory and Ma'at's quality governance are genuine innovations. But the first session of every sprint still loads the full canon — 254K tokens of everything.

Tiling solves the last mile. It makes session 1 nearly as efficient as sessions 2–5. It makes Thoth's context stable instead of slowly growing. It makes waste decline instead of slowly increase. And it introduces the manifest — borrowed directly from GPU tile lists — so the agent always knows what exists beyond its viewport.

94% efficiency versus 83% without tiling, versus 36% vanilla. But the real impact is qualitative: tiling prevents catastrophic sessions, eliminates technical debt from context-poor coding, and removes the need for defensive rules that accumulate in vanilla workflows.

Pantheon made AI-assisted development fast. Tiling made it efficient. Together, they deliver more productive output per dollar than any other approach we've measured — and the gap widens with every developer you add.
