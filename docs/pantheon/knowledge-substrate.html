Three Sovereignties · No Overlap

The knowledge substrate is one of three knowledge artifacts every Sirsi repo maintains. Each owns a different question. Each writes to a different file. The future hypergraph is the union.

$ /understand-anything:understand-dashboard # Open the interactive browser dashboard at 127.0.0.1:5173 — visual graph exploration.

$ /understand-anything:understand-explain internal/jackal/scan.go # Deep dive on a file with full neighbor context (imports, callers, tests).

$ jq '.layers[] | select(.id=="layer:core-services") | .nodeIds[]' .understand-anything/knowledge-graph.json # Direct JSON query — no AI required.

The Switch Within Sirsi Overall

Feature-Flagged Capability

The knowledge substrate is gated by a single config file and a build tag. Opt in per repo, opt out per fleet binary, propagate the same shape across every Sirsi product.

# configs/hypergraph.yaml
enabled: true                # Master switch
auto_refresh:
  on_commit: false           # Run /understand after every commit
  on_branch_switch: true     # Refresh on long-lived branch switches
exclude:                     # Mirrors .understand-anything/.understandignore
  - "docs/case-studies/**"
output_language: en
sync:
  thoth: true                # Auto-update .thoth/memory.yaml + journal.md after refresh
  hcs_publish: false         # Future: publish events to Hedera Consensus Service
hcs:                         # Future, inert until projection layer exists
  network: testnet
  topic_id: ""

When enabled: false, every sirsi hypergraph * command is a no-op. When the binary is compiled without the hypergraph build tag, the subcommand is absent entirely — no runtime cost in binaries that don't need it.

How To

Step 1

Install the prerequisite. The Understand-Anything plugin needs pnpm ≥ 10 and Node ≥ 22.

brew install pnpm
Step 2

Add the Claude Code plugin marketplace and install. One-time setup per machine.

/plugin marketplace add Lum1104/Understand-Anything
/plugin install understand-anything
/reload-plugins
Step 3

Run /understand on your repo. First run builds the full graph (~30 min on a 900-file repo). Subsequent runs are incremental (~30s for typical commits).

cd ~/Development/your-repo
# In Claude Code:
/understand
Step 4

Open the dashboard. Auto-launches at the end of /understand. Bookmark the full URL with the ?token= query string.

/understand-anything:understand-dashboard
Step 5

Query the graph. Three modes: AI chat, deterministic jq, or the dashboard's visual explorer.

/understand-anything:understand-chat
# or
jq '.nodes | map(select(.type=="function")) | length' \
   .understand-anything/knowledge-graph.json
Step 6

Keep it fresh. Re-run after meaningful changes. Incremental refresh uses the fingerprint baseline written on first run.

/understand              # Incremental (default after first run)
/understand --full       # Force full rebuild

FAQ

Does this replace Thoth or Seba?
No. Three sovereignties, three jobs. Thoth owns memory and intent. Seba owns deity-canonical architectural topology. The knowledge substrate owns auto-derived semantic verification. They sit at different abstraction levels and answer different questions.
Should I commit .understand-anything/ to git?
Solo dev: commit it. Team >3: gitignore it and rebuild in CI. The artifact is 1–3 MB JSON per repo and diffs render meaningfully, but it's also fast enough to rebuild that it doesn't need to be a tracked artifact.
Does it phone home?
No. The plugin runs entirely locally. The dashboard binds to 127.0.0.1 only. The graph never leaves your machine unless you explicitly commit or publish it.
What languages have function-level extraction?
C, C++, C#, Go, Java, JavaScript, PHP, Python, Ruby, Rust, TypeScript — via bundled tree-sitter grammars. Swift and Kotlin are file-level only in v2.7.5; function-level support requires an upstream parser add.
What if the upstream plugin gets abandoned?
The JSON is the contract, not the tool. The schema is documented in the plugin's repo and in ADR-019. If the plugin disappears, the JSON survives and can be regenerated by any replacement that targets the same schema. Worst case: fork the plugin — it's open source.
Will this work on non-Sirsi repos?
Yes. The plugin and the JSON format are tool-agnostic. You can run /understand on any codebase. The Thoth ↔ Substrate sync only activates in repos that have a .thoth/ directory.
Why Hedera and not Ethereum / Avalanche / IOTA?
Hedera Consensus Service fits the profile: aBFT finality in 3–5 seconds, fractions of a cent per message, optimized for high-volume / low-stakes-per-event workloads. Knowledge events are exactly that. Cylton's prior company Zimbali Networks received a $500K Hedera grant, so operational knowledge of the SDK and economics exists in-house. Not religiously locked — the architectural primitive needed is "ordered, durable, multi-writer event log." Hedera is the strongest current candidate.
